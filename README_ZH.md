# Skill Router

[English](README.md) · [中文版](README_ZH.md)

> 将相互重叠的 Agent Skill 收敛为聚焦、灵活的能力入口，同时保留原有能力。

`skill-router` 用于盘点已经安装的 Agent Skill，找出会竞争同类用户指令的高相似 Skill，并提出垂直 Router 方案。获得用户批准后，每个 Router 将成为更清晰的统一入口，根据具体任务选择一个叶子 Skill，或者协调多个叶子 Skill 共同完成任务。

## 它解决什么问题

当 Agent 只有少量、边界清晰的 Skill 时，通常能够正确选择。随着安装数量增加，容易出现：

- 多个 Skill 同时响应同一条自然语言指令；
- Agent 选择不稳定，或者重复加载相似说明；
- 相似 Skill 重复读取材料、调查问题或执行验证；
- 为减少入口而建立宽泛的“大 Router”，反而模糊能力边界。

`skill-router` 解决的是一个具体问题：**发现冲突**。如果同一条真实指令可能合理地触发多个 Skill，并且这些 Skill 追求相同的具体结果，它们之间就可能存在发现冲突。

属于同一领域、技术栈或工作周期，并不足以证明应该聚合。处理不同子目标的 Skill 可以在任务中协作，但不一定需要放进同一个 Router。

## 它如何工作

1. **盘点**：阅读已安装的 `SKILL.md`，理解每个 Skill 的触发意图、产出、独特能力、限制和依赖。
2. **比较**：使用真实用户指令，区分真正的选择冲突和宽泛的主题相似。
3. **规划**：向用户说明候选 Router、纳入与排除的 Skill、目录变化和回滚方式。
4. **批准后创建**：编写具体 Router，同时完整保留叶子 Skill 及其资源。
5. **验证**：测试单个 Skill、多个 Skill 配合、任务条件变化，以及不应触发 Router 的相邻请求。

在用户批准方案前，不会移动、隐藏或替换任何现有 Skill 入口。

## 灵活的能力协调

具体 Router 不会规定唯一、固定的工作流，而是说明：

- 叶子 Skill 共同处理的垂直用户意图；
- 每个叶子 Skill 提供的独特能力；
- 哪些任务信息会影响能力选择；
- 应当共享哪些上下文、证据和材料；
- 如何减少重复工作并整合结果。

Agent 再根据当前任务决定如何使用这些能力：

| 任务情况 | 可能的处理方式 |
| --- | --- |
| 一个能力已经足够 | 只使用一个叶子 Skill |
| 多个能力都有价值 | 选择相关能力共同参与 |
| 工作彼此独立 | 适合时并行处理 |
| 后续工作依赖前一步结果 | 按实际依赖进行协调 |
| 请求超出 Router 边界 | 改用保持独立的其他 Skill |

这些只是任务结构可能产生的结果，不是预设的调用模式。Router 不会为 Skill 固定主线、辅助、升级或兜底角色。

## 示例

假设多个调试 Skill 都可能响应：

> 诊断并修复这个偶发的测试失败。

`skill-router` 会先判断它们是否确实在解决同一个具体目标。如果成立，可以创建一个聚焦的调试 Router：

- 保留每个 Skill 独特的调查方法；
- 单个能力足够时只使用一个 Skill；
- 复现、根因分析和验证都有价值时让它们配合；
- 在选中的 Skill 之间共享证据和中间发现；
- 最终将不同结论整合为一份连贯结果。

如果相邻的测试规划 Skill 解决的是另一个目标，它仍然保持独立。因此，这个 Router 是垂直、具体的调试入口，而不是覆盖“软件开发”的大 Router。

## 什么时候使用

适合以下情况：

- 已安装的 Skill 太多，难以判断应该调用哪一个；
- 多个 Skill 会自然匹配同一种请求；
- 希望减少暴露入口，但不想删除或削弱能力；
- 希望相关 Skill 灵活配合，而不是固定一种组合；
- 已有 Router 过于宽泛、通用，或者固化了角色和顺序。

不要仅仅为了按主题分类，或者为了给整个领域建立一个统一入口而使用它。

## 安装

使用 Skills CLI：

```bash
npx skills add youaifuou/skill-router --skill skill-router
```

也可以在 [skills.sh](https://www.skills.sh/youaifuou/skill-router/skill-router) 查看市场页面。

手动安装时，克隆本仓库，并将 `skills/skill-router` 复制到 Agent 平台支持的 Skill 目录。

## 使用

可以向 Agent 提出：

> 盘点我已安装的 Skills，找出真正存在的发现冲突，并提出垂直 Router 方案。说明每个 Router 纳入哪些 Skill、哪些 Skill 应继续独立，以及入口将如何变化。先等待我批准，再执行修改。

本 Skill 会先汇报方案。用户批准后，才会创建和验证具体 Router，并执行双方确认的入口调整。

## 设计原则

- 默认保持 Skill 独立。
- 使用能够解决真实冲突的最小聚合范围。
- 从源 `SKILL.md` 提炼每项能力。
- 将详细方法和限制保留在叶子 Skill 中。
- 根据任务事实决定单独、组合、并行或依赖式协调。
- 保留原始目录、资源和回滚信息。

## 验证情况

本 Skill 已使用第三方 Skill 完成多轮隔离测试。生成的 Router 经过了单叶子请求、多叶子配合、授权边界、真实任务执行和相邻非触发场景验证。
